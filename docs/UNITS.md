# 新能源大基地规划模型 - 数据单位对照表 (Unit Conversion Table)

为了确保模型计算的准确性，本模型在输入配置（config.yaml）与内部计算（PyPSA/Optimization）之间采用了明确的单位转换逻辑。

## 1. 核心转换规则

PyPSA 模型默认使用 **MW (兆瓦)** 作为功率单位，**MWh (兆瓦时)** 作为能量单位。
然而，国内电力行业技经参数通常使用 **kW (千瓦)** 或 **kWh (千瓦时)**。

**转换公式:**
> 1 MW = 1,000 kW
> 1 MWh = 1,000 kWh

因此，涉及造价（CAPEX/OPEX）和燃料成本时，所有 `元/kW` 或 `元/kWh` 的输入值必须乘以 **1000** 转换为 `元/MW` 或 `元/MWh`。

## 2. 详细单位对照表

| 参数类别 | 参数名称 | 配置文件 (config.yaml) 单位 | 配置文件数值示例 | 模型内部 (PyPSA) 单位 | 转换系数 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **装机容量** | 风电/光伏/火电容量 | MW | 4000 | MW | 1.0 | 容量单位保持一致 |
| **储能容量** | 储能能量容量 | MWh | - | MWh | 1.0 | 容量单位保持一致 |
| **投资成本** | 风/光/火/储功率造价 (CAPEX) | **元/kW** | 3300.0 | **元/MW** | **x 1000** | 模型自动转换 |
| **投资成本** | 储能能量造价 (Energy CAPEX) | **元/kWh** | 750.0 | **元/MWh** | **x 1000** | 模型自动转换 |
| **运维成本** | 固定运维费 (Fixed OPEX) | **元/kW/年** | 30.0 | **元/MW/年** | **x 1000** | 模型自动转换 |
| **燃料成本** | 火电煤耗成本 | **元/kWh** | 0.185 | **元/MWh** | **x 1000** | 模型自动转换 |
| **惩罚成本** | 弃电惩罚 (Curtailment) | **元/MWh** | 100.0 | **元/MWh** | 1.0 | **无需转换** (本身即为 MWh) |
| **电力负荷** | 系统负荷 | MW | - | MW | 1.0 | - |

## 3. 代码实现位置

*   **config.yaml**: 用户输入界面，采用行业常用单位 (元/kW)。
*   **optimization_model.py**: `build_model` 函数中读取配置后乘以 1000。
*   **validator.py**: `calculate_economics` 函数中读取配置后乘以 1000。

## 4. 常见误区提示

*   **切勿重复转换**: 如果您直接在 config.yaml 中填入 3,300,000 (元/MW)，模型会再次乘以 1000，导致造价错误放大。请严格保持 config.yaml 使用 **元/kW**。
*   **弃电惩罚**: 弃电惩罚通常设定为较低值（如 0-100 元/MWh），其单位已经是 MWh，不需要乘以 1000。
